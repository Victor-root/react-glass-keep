# 🤝 Contributing to GlassKeep Enhanced

First of all: thank you 🙌
Issues, ideas, feature requests, bug reports and pull requests are all very welcome.

This project evolves a lot, and I am **very open to improvements and community PRs**, so please do not hesitate to contribute.

## 🐛 Reporting bugs

Please open an issue with **as much detail as possible** so the problem can be understood and reproduced quickly.

Helpful details include:

* a clear title
* what actually happened
* exact steps to reproduce
* screenshots or screen recordings if relevant
* the platform where the problem happens:

  * Web / PWA / Android / Android TV
  * Desktop or mobile
  * OS / browser / device if relevant
* whether the problem happens:

  * all the time
  * sometimes
  * only after a specific action
* logs / console errors / server errors if available

The more precise the report is, the easier it is to fix.

## 💡 Requesting features or improvements

Improvement requests are welcome too.

Please try to explain:

* the problem or limitation you are facing
* the improvement you would like
* why it would be useful in real usage
* whether it is mainly for desktop, mobile, TV, self-hosting, offline usage, etc.

Concrete use cases are always better than very short requests.

## 🔀 Pull requests

PRs are welcome, but please make them **clean, tested, and easy to review**.

### Please try to include:

* a short **Summary**
* what the PR changes
* why the change is useful
* screenshots / videos for UI changes when relevant
* **Closes #XXX** or **Related to #XXX** when the PR addresses an issue
* a small **Testing** section explaining what you tested

### Important expectations

* the change should be **tested and stable before opening the PR**
* please avoid PRs that are still very experimental or obviously broken
* if possible, keep the PR focused on one main topic
* make the description pleasant to read — a well-presented PR is much easier to review

## ✅ Testing

Before opening a PR, please test your change as much as you can.

Depending on the change, that may include:

* desktop
* mobile
* different browsers
* Android / Android TV
* offline / local-first behavior
* install / update flows
* regression checks on existing features

If you tested something specific, mention it clearly in the PR.

## 💬 Communication

If you are not sure about something, feel free to open an issue first and ask.

Even small ideas, UX suggestions, and quality-of-life improvements are welcome.
If you want to contribute, you are encouraged to do so 🚀

Thanks again for helping improve GlassKeep Enhanced ❤️
